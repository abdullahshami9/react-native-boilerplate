# User Stories & Requirements

This document outlines the core user stories for the **Raabtaa** ecosystem, categorized by user persona.

## 1. Individual User (Customer / Professional)
**Goal**: To discover services/products, network with others, and showcase professional skills.

### Discovery & Networking
- [ ] As an **Individual**, I want to **search for products and services** so that I can find what I need easily.
- [ ] As an **Individual**, I want to **follow businesses** (Business Purpose) and **connect with other individuals** (Social Networking).
- [ ] As an **Individual**, I want to **initiate chats** with businesses to ask questions or negotiate (if enabled by the seller).

### Commerce & Services
- [ ] As an **Individual**, I want to **add items to a cart and checkout** using **Cash on Delivery (COD)** or digital wallets (**Nayapay/EasyPaisa**).
- [ ] As an **Individual**, I want to **book appointments** with service providers (either **Auto-Approved** or **Approval-Based** depending on the provider's setting).
- [ ] As an **Individual**, I want to **view my order history** to track my purchases.

### Profile Management
- [ ] As an **Individual**, I want to **showcase my skills and education** on my profile, which is **Public by default** but can be toggled to **Private**.
- [ ] As an **Individual**, I want to **upload a resume** so that others can view my professional background.

---

## 2. Business User (Product-Based)
**Goal**: To manage inventory, sell products, and track sales performance.

### Inventory Management
- [ ] As a **Product Seller**, I want to **add product variants** (e.g., Sizes, Colors) so customers can choose the exact item they want.
- [ ] As a **Product Seller**, I want to **configure delivery fees**: set a default fee or toggle "Free Delivery" per product.
- [ ] As a **Product Seller**, I want to **define return policies** per product (Enable/Disable Returns) to manage expectations.
- [ ] As a **Product Seller**, I want to **set wholesale pricing tiers** (e.g., "Buy 10-50 for $5/unit") to attract B2B buyers.
- [ ] As a **Product Seller**, I want to **receive low stock alerts** (e.g., when stock < 5) to replenish inventory in time.

### Order Processing
- [ ] As a **Product Seller**, I want to **view incoming orders** (Pending status).
- [ ] As a **Product Seller**, I want to **update order status** (e.g., to Completed) once delivered.
- [ ] As a **Product Seller**, I want to **see a daily sales report** to understand my revenue.

### B2B Procurement
- [ ] As a **Product Seller**, I want to **view a procurement summary** that aggregates all pending items I need to fulfill orders.

---

## 3. Business User (Service-Based)
**Goal**: To manage service offerings, schedule, and appointments.

### Service Management
- [ ] As a **Service Provider**, I want to **list my services** with flexible types: **Hourly**, **Day/Night Shifts** (with diff prices), or **Multi-Day** (e.g., Workshops).
- [ ] As a **Service Provider**, I want to **define service location**: **On-Site** (Customer visits me) or **Home Service** (I visit customer).
- [ ] As a **Service Provider**, I want to **manage staff members** so customers can book specific staff with their own availability slots.
- [ ] As a **Service Provider**, I want to **configure cancellation policies** (e.g., 24-hour window) to protect my schedule.

### Appointment Management
- [ ] As a **Service Provider**, I want to **view my daily/weekly schedule**, including shift-based bookings (e.g., Marriage Hall Day Shift).
- [ ] As a **Service Provider**, I want to **accept or reject** appointment requests manually (if not set to auto-approve).
- [ ] As a **Service Provider**, I want to **block unavailability**, effectively preventing bookings on specific days or times.
